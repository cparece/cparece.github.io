CS Framework

A super lightweight framework to create simple content websites.

Getting Started

1. Download
There are two easy ways to get started! You may download the files or clone from github.

Download: Click here to begin your download.

Github: git clone “https://github.com/cparece/cparece.github.io/blob/master/styles.zip”
2. Setting Up
Once your files are unzipped and you have added them into your project folder be sure that the file <styles.css> is located in your css folder. In the header of your HTML pages, be sure to include the following head:
<link rel="stylesheet" href="css/styles.css">
3. Check out Examples
Once you get started, check out an example page to see how to combine components to create a super simple webpage.

Grid Layout

1 1 1 1 1 1 2 2 2 3 3 4 2 5 1 6
This framework utilizes a six column responsive grid system by default, but is customizable within the scss. The grid system is very simple and easy to use. When wanting to use the grid system, start with a wrapper div using the following classes: cs-container. Within the wrapper, you then can add how wide you would like your responsive columns by using the classes: cs-span-*. On smaller devices, the columns will transition into a one column layout by default.

Additionally, you can add breakpoints to elements so that you can control the different column breakpoints. Using classes small:
Size
Class
Examples
Small
cs-span-#-sm
cs-span-6-sm will span all six columns only when at the small breakpoint.
Medium
cs-span-#-m
cs-span-6-sm will span all six columns only when at the medium breakpoint.
Large
cs-span-#-lg
cs-span-6-sm will span all six columns only when at the large breakpoint.

Breakpoint sizes can easily be adjusted within the theme.scss file.
Example:
This div will span across four columns. Then will change to three columns at the m breakpoint. Then change to one column at the sm breakpoint.
This div will span across two columns. Then will change to three columns at the m breakpoint. Then change to one column at the sm breakpoint. 
            
<div class=”cs-container”>
    <div class=”cs-span-4”>
        This div will span across four columns.
    </div>
    <div class=”cs-span-2”>
        This div will span across two columns.
    </div>
</div>

Buttons

Buttons are elements that should clearly communicate the action that will occur when the user interacts with it.

There are a variety of buttons types that can be utilized. A good rule of thumb is to only allow one primary button per page. The rest of the buttons should be secondary or tertiary actions so that the the user has clear guidance on what the main call to action is on the page. Adding a raised primary button can help add additional context and draw the eye to the action the user should take.

Create a button using the button tag, then add any additional attributes to the class. Be sure to include the text caption, icon or image, and any additional event handlers and attributes.
<button class="cs-btn">Button</button>

Cards

Cards can be used to group related content together. They work well with utilizing the grid layout as a baseline than adding different size cards within the layout. To create a consistent experience, cards fill the height of the other elements within the grid row.

Create a card using the card tag, then add any additional attributes to the class. Be sure to include the grid span size and any additional attributes.

Within a card, the following elements can be used:
  • cs-title
  • cs-body
  • cs-actions
            
  <div class="cs-container">
    <div class="cs-card cs-span-3">
      <div class="cs-title">Card Title</div>
      <div class="cs-body">Card Body</div>
    </div>
  </div>

Navigation

The navigation is a basic horizontal navigation that will turn into a vertical navigation for the small breakpoint. To utilize the navigation component you must use the nav tag along with the class cs-nav. The navigation can also be fixed at the top by utilizing the class fixed-top. To highlight the selected navigation item, add the class active to the <li>.

Table
